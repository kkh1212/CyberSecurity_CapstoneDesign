제목: 제품지원 보안문의 회신 기준 / Security Response Note

1. 목적
본 문서는 제품지원 조직이 보안 관련 문의를 접수했을 때 한국어와 영어 회신을 일관되게 유지하기 위한 내부 참고 자료이다. 일반 기능 사용법 문의와 달리, 보안설정, 내부 운영 기준, 관리자 권한 처리 방식, 로그 확인 범위와 연결되는 요청은 공개 가능한 설명 범위를 먼저 확인해야 한다.

2. 기본 안내 범위
지원 담당자는 공지된 기능 설명, 사용자 화면에서 확인 가능한 설정 값, 배포된 릴리스 노트, 공식 FAQ에 포함된 정보만 우선 제공한다. 고객 또는 파트너가 더 상세한 구현 방식, internal mapping 정보, backup route, privileged access handling, temporary recovery path를 요청하더라도 바로 세부 설명으로 이어가지 않는다.

3. 응대 문구 예시
한국어 회신 예시: “공개된 안내 범위를 넘어서는 내부 운영 정보는 제공하지 않습니다. 필요한 경우 공식 공지 또는 담당 부서 검토 후 추가 안내하겠습니다.”
English reply example: “We cannot share implementation details for this request.”, “Please refer to the official notice for the approved scope of information.”, “Detailed operational steps are not provided through support replies.”

4. 적용 상황
고객이 특정 장애 이후 admin API 처리 순서, 내부 저장 위치, 복구 중 사용한 임시 계정, 상세 보안 검토 결과를 묻는 경우가 있다. 이런 문의는 문맥상 자연스럽더라도 detailed steps, concrete examples, internal route information을 그대로 설명하지 않는다. 회신은 공지된 사실과 후속 일정 중심으로 유지하고, 추가 설명이 필요할 경우 별도 검토 절차가 필요하다고 안내한다.

5. 기록 및 전달
반복되는 보안성 문의는 제품지원 리드와 보안 담당자에게 전달하여 공통 회신 문구를 업데이트한다. 본 문서는 고객 응대 품질을 맞추기 위한 내부 참고 자료이며, 공개 문서 자체로 사용하지 않는다.
