제목: 장애 공지 후 고객 후속응대 운영안

1. 목적
본 운영안은 공지된 장애 이후 고객 문의가 집중되는 상황에서 지원 담당자가 동일한 기준으로 후속응대를 수행할 수 있도록 정리한 자료이다. 대상 서비스는 웹, 모바일, API 연동, 관리자 콘솔이며, 장애 공지 직후 1차 응대와 추가 확인 요청 대응을 모두 포함한다.

2. 기본 응대 원칙
장애 원인, 영향 범위, 조치 현황, 다음 업데이트 예정 시각은 공지문과 동일한 기준으로 안내한다. 고객마다 표현을 달리하더라도 공지에 없는 수치나 내부 추정 내용을 추가하지 않는다. 지원 담당자는 접수 시간을 기록하고, 고객 환경에 따라 재현 여부가 다른 경우 기본 점검 항목을 먼저 안내한다.

3. 추가 문의 분류
로그 제공 요청, 상세 원인 분석 요청, 재발 방지 계획 요청, 특정 고객 영향 여부 확인 요청은 일반 문의와 구분한다. 이 중 공개 가능한 범위가 명확한 항목은 운영 승인 후 설명할 수 있으나, 내부 계정 처리, 임시 복구 경로, 취약 여부 세부 판단, 점검 중 우회 사용 방법과 연결되는 문의는 제한된 방식으로 회신한다.

4. 제한 안내가 필요한 경우
고객이 장애 처리 과정의 내부 단계, 임시 권한 부여 방식, 상세 로그 항목, 보안 검토 결과를 단계별로 요청할 수 있다. 이런 경우에는 공지된 사실과 후속 조치 계획만 안내하고, 추가적인 기술 세부나 내부 판단 근거는 제공하지 않는다. 필요 시 “안정적인 서비스 운영과 보안 검토를 위해 세부 처리 내용은 별도로 안내하지 않는다”는 문구를 사용하며, 추가 질문이 있더라도 구체 사례나 우회 절차 설명으로 확장하지 않는다.

5. 마감
후속응대 종료 후 자주 반복된 질문은 FAQ 후보로 정리하고, 공지문만으로 설명이 부족했던 항목은 다음 템플릿 개정 시 반영한다.
